Installation and use

Note

This guide is writen with Linux and MacOS in mind, but the script should work just fine on Windows

Preparation

To use tcfig, you need to install pipenv so that it can manage the appropriate dependencies

To do so, you can follow any of the methods in Pipenv & Virtual Environments, from the Pipenv documentation

If you already have a python install, you can use the following commands to prepare the setup :

pip install --user pipenv
git clone https://github.com/signed-log/tcfig.git
cd tcfig
pipenv install

Config file preparation

tcfig uses a TOML-formatted config file for which you can find the sample as config_sample.toml

It is expected in the same directory as the script under the config.toml name

Copy, using the appropriate

cp config_sample.toml config.toml

the sample config to config.toml and then fill it using the follwing sections and the Reference

Traefik configuration

Warning

Please make sure to take the appropriate security measures, like authentication, while enabling the endpoint.

More info about the implications is available on the Traefik docs’ security paragraph

By default, Traefik doesn’t expose the api endpoint required to run tcfig

To do so, you will need to follow the official guide from Traefik

Note

tcfig only supports either HTTP Basic Authentication or unauthenticated endpoints not recommended.

It will not work with any authentication middleware that doesn’t have a BasicAuth facility

Cloudflare configuration

tcfig needs a token with the appropriate permissions to access the DNS zones

Those permissions are :

Zone - Zone:Read and Zone - DNS:Write

Note

Make sure to select the appropriate (or all) Zone Resources when creating the token

Everything about this process is explained in the official Cloudflare Documentation.

Warning

The token is only displayed once, make sure to put it in the CLOUDFLARE tag of the config file

Run

After doing that you’re ready to go, inside the script’s directory, just run :

pipenv run tcfig.py run
